# 実装計画

機能や改修の着手前に、実装の方針とタスクをここにまとめます。レビューや進捗確認のたびに参照できるドキュメントです。

## 使い方

1. `_template.md` をコピーする
2. ファイル名は **機能・課題が分かる名前** にする（例：`password-reset.md`、`issue-36-implementation-plan-template.md`）
3. プレースホルダー（`[...]`）を埋め、不要なセクションは削除する
4. **Summary (EN)** を英語で記入する（本文は日本語のままでよい）
5. 実装が進んだら **§3 タスク一覧** のチェックリストと **更新履歴** を更新する
6. 関連する GitHub Issue / PR があれば、計画の先頭にリンクを書く

## 計画の更新タイミング

| タイミング | 更新すること |
| --- | --- |
| **レビュー依頼前** | ステータス →「レビュー中」、関連 PR リンク、今回 PR で完了したタスクを `[x]`、更新履歴に 1 行 |
| **レビュー中** | 方針が変わったら本文 or 更新履歴を随時修正 |
| **マージ前** | チェックリスト最終反映、計画と実装の差分があれば更新履歴に記載、全完了ならステータス →「完了」 |

機能全体を一度に実装しない場合、未着手タスクは `[ ]` のままで問題ありません。

## 含める内容（Issue #36 より）

| セクション | 目的 |
| --- | --- |
| Summary (EN) | 海外向けに Goal / Approach / 受け入れ条件の要点（短く） |
| 目的・背景 | なぜやるのか、前提・制約 |
| 実装概要 | 何を、どのように実装するか（使用パッケージは新規依存がある場合のみ） |
| タスク一覧 | パッケージ別チェックリスト（進捗追跡） |
| 更新履歴 | PR 単位のマイルストーン・方針変更の記録 |
| 実装詳細 | 新規ファイル・クラス、既存ファイルへの API 追加など（レビュー用） |

## 言語

- **本文**: 日本語（チーム内のレビュー・実装用）
- **Summary (EN)**: 英語（ポートフォリオ、海外クライアント、GitHub 上で要点だけ伝えたいとき）
- 計画全体の英訳は不要。先頭の英語サマリーだけで十分なことが多い
